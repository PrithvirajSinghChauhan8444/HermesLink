# Changes Detail - 2026-03-03

## Bug Fix: Downloads Submitted from Web Never Executed

**Root Cause:** `api_server.py` created a `JobManager` instance but never started a `JobRunner` — the background thread responsible for actually executing downloads. Jobs submitted via the web API were written to `jobs.json` in `PENDING` state and sat there indefinitely. The CLI (`controller.py`) worked because it explicitly starts a `JobRunner` alongside its `JobManager`.

**Fix applied to `api_server.py`:**
- Imported `JobRunner` and `JobController`.
- Added a FastAPI `lifespan` context manager that starts `JobRunner` on server boot and stops it cleanly on shutdown.
- Added a `DEFAULT_DOWNLOAD_DIR` (`~/Downloads/hermeslink_downloads`) that is used as the fallback `destination` when the client doesn't specify one — matching CLI behaviour.
- Added new endpoint `POST /api/jobs/{job_id}/action` accepting `{"action": "PAUSE|RESUME|STOP|RETRY|RESTART"}` so the frontend can control jobs.

## Bug Fix: `No module named 'src'` Errors in Engines and JobRunner

**Root Cause:** Several files used `from src.engines.xxx import ...` and `from src.core.models import JobState` — import paths that only resolve when the project root (parent of `src/`) is on `sys.path`. `api_server.py` only adds `src/` itself to `sys.path`, so `src` was not a recognisable module name at runtime.

**Files fixed (removed `src.` prefix from imports):**
- `core/job_runner.py` — engine imports: `from src.engines.xxx` → `from engines.xxx`
- `engines/aria2.py` — 2 occurrences: `from src.core.models` → `from core.models`
- `engines/direct.py` — `from src.core.models` → `from core.models`
- `engines/media.py` — `from src.core.models` → `from core.models`

## Feature: Pause / Resume / Stop Buttons on Active Downloads

Added per-job control buttons to the `ActiveJobsSection` in the frontend.

**`services/api.js`:**
- Added `endpoints.jobs.action(id, action)` — calls `POST /api/jobs/{id}/action`.

**`ActiveJobsSection.jsx`:**
- Added `handleAction(jobId, action)` that calls the API, disables the button while in-flight, and immediately refreshes the job list on completion.
- Button visibility is state-aware:
  - `RUNNING` → **Pause** + Stop
  - `PAUSED` → **Resume** + Stop
  - `PENDING` → Stop only
- Added a `status-pending` indicator dot (grey, pulsing) for PENDING jobs.
- Filename display falls back to URL if no filename is available yet.

**`ActiveJobsSection.css`:**
- Added `.job-controls`, `.ctrl-btn`, `.ctrl-pause` (amber), `.ctrl-resume` (green), `.ctrl-stop` (red), `.status-pending` styles.
