# Documentation of Changes – 2026-04-03

## 1. yt-dlp Format Selection and Pre-fetching
* **Files Modified:** `backend/src/api_server.py`, `frontend/HermesLink_frontend/src/components/features/NewJobModal.jsx`
* **Details:** Added a formal inspection flow for YouTube and custom media downloads. Implemented a new asynchronous `GET /api/yt-dlp/info` endpoint acting as a secure bridge to execute fast `yt-dlp --dump-json` native inspection queries.
* **Frontend UI Implementation:** Extensively rebuilt the `NewJobModal.jsx` download form to feature a direct "Fetch Formats" interface. The extracted underlying formats are automatically filtered into organized `<optgroup>` layout clusters (e.g., "Audio Only", "Video + Audio", "Video Only (No Sound)"), completely neutralizing the risk of picking silent or corrupt streams randomly while keeping the interface extremely user-friendly.

## 2. Granular byte-level Metrics Extraction
* **File Modified:** `backend/src/engines/yt_dlp.py`
* **Details:** Drastically enhanced the core `regex` STDOUT parser module inside `YTDLPEngine.start`. Wrote an internal size parser (`_parse_size_to_bytes`) to reliably detect, cast, and map arbitrary network strings into absolute strictly typed integers.
* **Outcome:** The yt-dlp engine formally propagates variables like `completed_length`, `total_length`, and `eta` natively over the RTDB payload framework. This inherently fixes a recurring visual bug where yt-dlp jobs displayed a stagnant "0 B / 0 B" downloaded progress metric inside `ActiveJobsSection.jsx`.

## 3. Strict Playlist Negation Defaults
* **Files Modified:** `backend/src/engines/yt_dlp.py`, `backend/src/api_server.py`
* **Details:** Safely patched widespread latency limitations tied to URL syntax edge-cases. Enforced rigorous `--no-playlist` and `--no-warnings` CLI flags natively across all executing boundary points (both API inspection parsing and core downloads). This effectively limits the yt-dlp thread scope strictly to individual targeting, avoiding complete network collapse if a standalone URL additionally contains a subtle `&list=` playlist parameter wrapper.
