# 2026-01-23

## Job Runner Implementation

**Branch:** `QUEUE`

### Solved Error

Fixed an issue where batch downloads (multiple links provided at once) failed to start. Specifically, while the jobs were correctly created and queued, the download engine was not being initialized for them, causing them to hang in a "RUNNING" state without actual execution.

### Implementation Details

- **Created `src/core/job_runner.py`**:
  - Implemented a background service `JobRunner` that runs in a separate thread.
  - It continuously monitors the `JobManager` for the active `RUNNING` job.
  - Automatically instantiates and starts the appropriate download engine (`MediaEngine`, `DirectEngine`, or `Aria2Engine`) when a job becomes active.
  - Handles engine lifecycle (start, stop, cancel) and state synchronization.

- **Updated `src/controller.py`**:
  - Integrated `JobRunner` initialization into the `main()` loop.
  - Refactored `add_new_download` to remove manual engine startup logic. It now simply creates jobs and lets the `JobRunner` handle execution.
  - This decoupling ensures downloads continue running in the background even when navigating the CLI menus.
