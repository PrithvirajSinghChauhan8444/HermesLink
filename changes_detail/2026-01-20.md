# Changes: 2026-01-20

## Job Persistence & Stability (Aria2 Routing Support)

### 1. Atomic File Writes in `JobManager`

- **Problem**: The Dashboard was encountering empty files or partial reads while the Controller was writing `jobs.json`, causing flickering ("No jobs found").
- **Solution**: Implemented atomic writes using a "write-to-temp-and-rename" strategy.
  - Data is written to `jobs.json.tmp` first.
  - `os.replace` is used to atomically swap the temp file with the target file.

### 2. Windows File Locking Fix (Retry Logic)

- **Problem**: `WinError 5: Access is denied` occurred when `os.replace` tried to swap files while the Dashboard had `jobs.json` open for reading (a mandatory lock on Windows).
- **Solution**: Implemented an **Optimistic Retry** mechanism in `save_jobs`.
  - The system now catches `PermissionError`.
  - Retries the operation up to 5 times with a 0.1s delay between attempts.
  - This allows the Writer (Controller) to wait out the brief Read lock held by the Dashboard.

### 3. Dashboard Optimization

- Increased the main refresh loop interval from **1s** to **3s** to reduce disk I/O contention and visual flickering.
