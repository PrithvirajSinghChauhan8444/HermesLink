# Changes Detail - 2026-03-05

## Feature: Floating SVG Decorations

Added floating SVG decorations from the `assets` folder into various sections of the frontend application to improve aesthetics. SVGs depicting errors/bugs (from the `her3` folder) were intentionally omitted.

**Sections Updated:**
- `LandingSection` (using SVGs from `her1` and `her2 (2)`)
- `QueueSection` (added document and gear SVGs)
- `ActiveJobsSection` (added dynamic-feeling SVGs like fast-moving gears and progress arrows)
- `HistorySection` (added loading monitors and file stacks)
- `SettingsSection` (added gears and portal SVGs)

## Refactoring: SVG Theme/Opacity Management

Initially, SVGs were uniformly styled, but this caused visibility issues on varied backgrounds (dark vs white). The styling approach was refactored for better maintainability and visual contrast.

**Global CSS (`index.css`):**
- Introduced `.floating-svg-dark` utility class `filter: invert(0);` for SVGs on dark backgrounds (Landing, Queue, Active Jobs).
- Introduced `.floating-svg-light` utility class `filter: invert(1);` for completely inverting SVGs to white on light backgrounds (History, Settings).
- Removed `opacity` values from these global classes to avoid CSS specificity conflicts with component-level styles.

**Component-Specific CSS (`*Section.css` & `*Section.jsx`):**
- Restored `opacity` control to individual component stylesheets.
- Added scope-specific classes (e.g., `.floating-svg-landing`, `.floating-svg-queue`, `.floating-svg-active`, `.floating-svg-history`, `.floating-svg-settings`) to ensure `opacity` and `animation` rules do not leak or conflict across different sections of the application.
