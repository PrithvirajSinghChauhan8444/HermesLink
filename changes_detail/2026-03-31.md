# Documentation of Changes – 2026-03-31

## Phase 1 Engine Expansion: yt-dlp Integration
Today focused on expanding the HermesLink downloading architecture to support YouTube video processing natively using `yt-dlp`.

### 1. New Engine: `YTDLPEngine`
* **File Added:** `backend/src/engines/yt_dlp.py`
* **Details:** Created a new class inheriting from `BaseEngine`. It spawns `yt-dlp` as a background subprocess with `--newline` and monitors its `stdout` via a daemon thread.
* **Progress Tracking:** The thread parses percentage, download speed, and filenames in real-time, feeding them to Firebase via `JobManager.update_progress()`.
* **Throttling:** Implemented a 1-second throttle limit on Firebase updates to prevent the frontend UI from excessively flickering under high-frequency CLI outputs.
* **Control States:** Fully supports pausing and resuming via native Linux OS hooks (`signal.SIGSTOP` and `signal.SIGCONT`), avoiding the need to forcefully kill the process to pause.

### 2. Job Dispatcher Updates
* **File Modified:** `backend/src/test_agent.py`
* **Details:** Ripped out the hardcoded `Aria2Engine` instantiation. Introduced generic engine selection logic. Submissions detecting a `youtube.com/youtu.be` link, or having their `engine_config.type` set to `yt-dlp`, are automatically routed to the new `YTDLPEngine`.

### 3. Frontend UI Isolation
* **File Modified:** `frontend/HermesLink_frontend/src/components/features/NewJobModal.jsx`
* **Details:** Removed irrelevant placeholder options (`direct`, `media`, `p2p`) from the New Download modal block. Restricted engine selection strictly to `Aria2 (Default)` and `yt-dlp (YouTube/Media)`.

## Infrastructure Fixes & Enhancements

### 4. Added Missing Stats Endpoint
* **File Modified:** `backend/src/api_server.py`
* **Details:** Fixed an issue where `LandingSection.jsx` threw an Axios `404 Not Found` error when fetching dashboard widget stats. Added a new `GET /api/queues` endpoint that parses `job_manager.configs` to return the currently deployed queue configuration map.

### 5. Persistent Device Context 
* **File Modified:** `backend/src/test_agent.py`
* **Details:** Overhauled test isolation limits. The main worker agent no longer spins up a randomized UUID every time it boots. It now possesses an internal `_get_or_create_device_id()` method that writes the generated ID to `test_agent_config.json`. Consequently, jobs queued while the agent is offline map safely upon reboot.
