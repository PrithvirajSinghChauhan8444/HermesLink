# Changes: 2026-01-21

## Branch Management

- **New Branch**: Created a new branch named `QUEUE`.
- **Scope**: All changes detailed below and throughout the day were committed to this `QUEUE` branch.

## Job Queue and Control

- **Single Queue System**: Developed a robust `JobQueue` class ensuring only one job is active at a time, with automatic queue advancement and state persistence.
- **Job Controller Implementation**:
  - Refactored `src/core/controller.py` to `src/core/job_controller.py` to resolve naming conflicts.
  - Implemented `JobController` as the intermediary for job commands.
  - Defined a fixed command set (start, pause, etc.) and a command dispatcher with state-based validation.
  - Exposed a read-only status feed for the dashboard.
- **Dashboard Control**: Enabled control actions by Job ID through the new controller.

## Agent System Refactoring

- **Prompt Overhaul**: Refactored system prompts for all worker agents (System, Gmail, Calendar, Memory, WhatsApp).
  - Shifted from tool-prescriptive prompts to capability-focused descriptions.
  - Empowered agents to autonomously select appropriate tools based on their defined capabilities.
- **Tooling Fixes**: Resolved issues with tool naming (e.g., `send_whatsapp_msg`) and agent instantiation errors.
