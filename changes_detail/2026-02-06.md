## Linux Compatibility Fixes

The system is now **somewhat stable on Linux**. Multiple fixes were made to ensure the backend runs properly on Linux systems.

### Backend Fixes

- **REMOVED** Windows-only `import msvcrt` from `controller.py` - this module doesn't exist on Linux and wasn't being used.
- **ADDED** Missing `List` import to `core/models.py` - was causing `NameError: name 'List' is not defined`.
- **ADDED** Missing `Any` import to `engines/aria2.py` - was causing `NameError: name 'Any' is not defined`.
- **FIXED** Hardcoded Windows path `G:\STUFF\Watch\hermeslink_test_download` in `controller.py` - now uses cross-platform `~/Downloads/hermeslink_downloads`.

### Dashboard Improvements

- **FIXED** Non-blocking keyboard input for Linux using `tty.setraw()` with select timeout.
- **CHANGED** Key matching to use `.upper()` for case-insensitive input (M/m and Q/q both work).
- **ADDED** Enter key handling - pressing Enter now immediately refreshes the dashboard view.

### Aria2 Engine

- **VERIFIED** aria2c daemon starts and runs correctly on Linux.
- **TESTED** Download via aria2 engine completed successfully.

## Backend-Frontend Integration

Connected the Python backend job data to the React frontend, enabling live data display.

### Backend API (FastAPI)

- **NEW** `api_server.py` - REST API server with the following endpoints:
  - `GET /api/jobs` - All jobs
  - `GET /api/jobs/active` - PENDING/RUNNING/PAUSED jobs
  - `GET /api/jobs/history` - COMPLETED/FAILED/STOPPED jobs
  - `GET /api/jobs/{id}` - Single job details
  - `GET /api/queues` - Queue configurations
  - `POST /api/jobs/reload` - Reload from disk
- **ADDED** `fastapi` and `uvicorn[standard]` to `requirements.txt`
- **ENABLED** CORS for frontend dev server (ports 5173, 3000)

### Frontend Services

- **NEW** `api.js` - API client with fetch functions for jobs, history, queues
- **NEW** `useJobs.js` hook - React hook with:
  - Loading/error state management
  - Auto-refresh support (configurable interval)
  - Utility functions: `formatBytes`, `formatDate`, `extractFilename`

### Frontend Pages Updated

#### ActiveJobs Page
- Fetches active jobs from API with 3-second auto-refresh
- Displays job cards with: filename, progress bar, speed, state badge
- Loading skeleton, error display, empty state components

#### History Page  
- Fetches completed/failed/stopped jobs from API
- Stats summary showing count by state (Completed/Failed/Stopped)
- State badges with color coding (green/red/gray)
- Error reason display for failed jobs

### Layout Fixes

- **FIXED** Content overflow - all content now stays within glass card containers
- **ADDED** `max-height: 70vh` with `overflow-y: auto` for scrolling
- **ADDED** Custom styled scrollbars
- **ADDED** Text truncation with ellipsis for long filenames/errors

## How to Run

```bash
# Start Backend API
cd backend/src
source ../.venv/bin/activate
uvicorn api_server:app --port 8000

# Start Frontend
cd frontend/HermesLink_frontend
npm run dev
```

- Frontend: http://localhost:5173
- API Docs: http://localhost:8000/docs
