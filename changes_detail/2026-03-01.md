# Changes Detail - 2026-03-01

## Fixed Hardcoded Active State on Settings Cards
- The "Network" card in `SettingsSection.jsx` was permanently styled as active due to a hardcoded `index === 1` condition.
- Removed the condition — all four cards (`General`, `Network`, `Engines`, `Storage`) now share the same uniform class.
- Cleaned up unused `.settings-card-active` and `.card-title-active` classes from `SettingsSection.css`.

## Inverted Hover Theme for Settings Cards
- Updated `SettingsSection.css` to follow the project's inverted colour rule (dark background → white hover/active).
- On hover: card background → **white**, border → **white**, title text → **black**.

## Smooth Hover Transition on Settings Cards
- Extended Tailwind's default `transition-colors` (150ms) to `duration-500 ease-in-out` on the card, title, and description.
- Results in a polished, gradual black → white fade instead of an abrupt snap.

## Smooth Mouse Wheel Scrolling Between Sections
- Native browser scroll was hard-snapping between sections due to CSS scroll-snap.
- Added a `wheel` event listener in `MainLayout.jsx` that intercepts scroll and routes it through GSAP (`power2.inOut`, 1s duration) — same animation as navbar clicks.
- Added `isScrolling` ref as a lock to prevent scroll chaining mid-animation; unlocks on `onComplete`.

## ActiveJobsSection CSS Rebuild
- Restored and expanded `ActiveJobsSection.css` with a full set of semantic class definitions.
- Covers: container, header, stats, job grid, loading/empty states, job cards, progress bar, and footer.
- Progress bar includes a glow effect (`.progress-bar-glow`) and status indicators (`.status-running`, `.status-paused`).
